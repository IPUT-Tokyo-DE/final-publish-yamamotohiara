# ここにゲームのタイトルを記述する
motimoti
## ゲームの内容
空から降ってくるのは、思わず食べたくなるほど大きくてもちもちなおもち。
白くてつやつやな白もちや草もち、豆もちなど、見るだけでお腹がすく見た目。
プレイヤーは落ちてくるおもちをキャッチして集めるだけ。
直感的に遊べて、わくわく楽しい、軽快なパーティーゲーム。

## 操作説明
左スティックでキャラクターを移動します。

• どのような世界（構図や広さ)で
円形の直径11メートル程のステージ。ポップでもちもちな世界観。
• 何を
自分のキャラクターを
• どのように操作して
コントローラーの左スティックで移動させて
• 何をやるゲームか？ 
障害物を避けながら、上から降ってくるおもちを集める（串に刺す）ゲーム
• 面白い点は何か？
1.	見ているだけで食べたくなる「大きくて、もちもちなお餅」を好きなだけ集められるため、わくわく感と満足感を味わえる。
2.	簡単操作でテンポよく遊べるポップなゲームなので、家族や友達とワイワイ気軽に楽しめる。
3.	上からの視点で進行し、落ちてくる直前まで「障害物」か「お餅」か分からない演出により、どきどき・はらはらする高揚感を味わえる。
• どうなったら成功か？
一番多くお餅を集めた人が勝ち
• どうなったら失敗か？
失敗はないが、お餅の数に応じて順位がつく

2. アピールポイント

技術的に工夫した点
タイトルシーンからゲームシーンへ遷移する際に、プレイヤーの情報を引き継ぎ、各シーンの PlayerInputManager に渡す仕組みを実装した。これにより、シーンをまたいでも入力設定やプレイヤー情報が正しく維持されるようになっている。
お餅と障害物が落下する割合について、Inspector上で調整可能なパラメーターを作成した。スクリプトを直接編集することなく数値を変更できるため、ゲームバランスの調整を効率的に行えるようにした。
Unityの物理的な影表現は使用せず、床に投影される「影の絵」を別途用意し、落下物が地面に近づくにつれて影が大きくなる演出を実装した。

ゲームの面白さを高めるために工夫した点
お餅と障害物（岩）が落下する際、最初に表示される影の大きさを同一に設定した。これにより、落下物が地面に近づく直前まで正体が分からず、プレイヤーに判断の緊張感を与える工夫をしている。
全体をポップで親しみやすいデザインにすることで、年齢やゲーム経験を問わず遊びやすい雰囲気を意識した。
操作方法を移動のみに限定し、複雑な入力を必要としない設計にすることで、誰でも直感的にプレイできるようにした。
一人では取り切れない量のお餅が落下するように調整し、複数人で遊ぶ際に競争や盛り上がりが生まれるよう工夫した。

 4. AI利用報告
使用したAI：Gemini
コードの生成とエラー修正: 「影を出す」「岩を降らせる」といった具体的な機能
数学的な計算の代行: 「円形ステージの外に出ないようにするベクトル計算」や「高さに応じた影のサイズ変更（Lerp）」など
アイデアの具体化: 「お餅と串」というコンセプトから、それをどうUnityで実現するか（親子関係にするか、アニメーションさせるかなど）の選択肢の提案

ゲームとしての仕組みはうまくいったが、見た目や効果音はうまくいかなかった。
AIを使うことでスムーズに開発が進んだが、タグのつけ忘れなどの簡単なミスは逆に混乱してしまった。
